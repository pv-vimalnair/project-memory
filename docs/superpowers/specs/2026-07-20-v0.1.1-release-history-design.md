# Project Memory v0.1.1 Release History Design

## Goal

Make Project Memory v0.1.1 the current installable plugin release without losing or rewriting the v0.1.0 history.

## Considered approaches

1. **Preserve releases with Git tags and release artifacts (selected).** Keep v0.1.0 immutable, merge the approved v0.1.1 work into `main`, and create a distinct v0.1.1 tag and release artifact. This preserves complete history while giving users one current installable version.
2. **Maintain permanent version branches.** Keep separate long-lived branches for v0.1.0 and v0.1.1. This adds maintenance and merge overhead without improving release recovery beyond tags.
3. **Replace v0.1.0 in place.** Update the existing release to contain v0.1.1. This would obscure published history and is rejected.

## Approved design

- Preserve the v0.1.0 tag, release records, authorization evidence, and artifacts unchanged.
- Merge `codex/post-bootstrap-history-import` into local `main` without squashing or rewriting its commits.
- Treat v0.1.1 as the current source and installable release candidate.
- Create the v0.1.1 tag and publish its release only during a separately authorized publication step.
- Do not delete v0.1.0, retarget its tag, alter its artifacts, or force-push history.
- Keep the feature branch and worktree until the merge and verification are complete.

## Validation

- Confirm v0.1.0 still resolves to its original commit and remains recoverable.
- Confirm `main` contains commit `af623d9bca914b13c7ab786bddc7fc481300f7c1` after the merge.
- Confirm packaged plugin metadata identifies v0.1.1 as the current version.
- Run the full Project Memory test suite on merged `main`.
- Confirm the merged checkout is clean.
- Do not push, tag, install, or publish during the local merge step.

## Failure handling

- If `main` is dirty, stop before merging.
- If the feature branch cannot be fast-forwarded cleanly, stop and review the divergence rather than creating an unplanned merge commit.
- If validation fails, keep v0.1.0 untouched and do not tag or publish v0.1.1.
