# Security Policy

## Supported versions

The latest `0.1.x` release receives security fixes. Older development snapshots
are not supported.

## Report a vulnerability privately

Use [GitHub private vulnerability reporting](https://github.com/pv-vimalnair/project-memory/security/advisories/new).
Do not put vulnerability details, credentials, tokens, private project content,
or exploit instructions in a public issue.

Include the affected version, the minimum reproducible detail, likely impact,
and any safe mitigation you have already tested. Remove secrets and personal or
proprietary data from evidence.

The maintainer will acknowledge a valid report, investigate it, coordinate a
fix, and publish an advisory when appropriate. Do not test against systems or
repositories you do not own or have explicit permission to assess.
